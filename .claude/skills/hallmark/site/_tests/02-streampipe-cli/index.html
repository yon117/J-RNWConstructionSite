streampipe v0.4.1 · MIT · 1.2 MB

$streampipe — a small, fast, single-binary CLI for parsing log and event streams from stdin.

Streampipe reads newline-delimited records (JSON, logfmt, CSV, raw) from stdin, applies a sequence of filters and transforms, and writes the result to stdout. Composes with anything that emits lines — tail -f, kafkacat, journalctl, kubectl logs. No daemon, no config file, no telemetry. 

$ tail -f access.log | streampipe parse --format nginx --filter 'status >= 500' --out json    {"ts":"2026-04-30T14:01:18Z","ip":"203.0.113.42","method":"POST","path":"/api/v1/checkout","status":502,"ms":4101}{"ts":"2026-04-30T14:01:21Z","ip":"203.0.113.78","method":"GET","path":"/api/v1/orders/9128","status":503,"ms":12}{"ts":"2026-04-30T14:01:21Z","ip":"203.0.113.42","method":"POST","path":"/api/v1/checkout","status":502,"ms":3998}

Streampipe is written in Rust, ships as a 1.2 MB static binary, and is designed to be the last process in your pipeline before storage. It does not buffer. It does not retain state. It does one thing per invocation, and you can pipe it into another invocation.

02   How it works3 notes

1.0

Stream in, stream out. Streampipe reads stdin line by line and writes stdout line by line. Each record is independent. Memory use is bounded by the largest record, not by the size of the stream. You can pipe a 200 GB log file through a 1.2 MB binary.

2.0

Filters are expressions, not config. The --filter flag takes a CEL-style expression evaluated against each parsed record: status >= 500, ms > 1000 && path.startsWith('/api'), !has(headers.x-debug). No filter file. No filter language to learn beyond the expression.

3.0

Transforms are projections, not migrations. The --map flag rewrites each record into a new shape: --map '{ts: ts, p99: ms, where: path}'. Records are immutable inside Streampipe; transforms produce new records. There is no in-place edit; debugging is therefore a pipe with one extra stage.

03   Flagsfull reference at streampipe.dev/docs

FlagTypeWhat it does
--formatnginx | apache | json | logfmt | csvParse stdin as the named format. Default: auto-detect from first 3 lines.
--filterexpressionDrop records that don't match. CEL-style. Strings, numbers, booleans, arrays.
--mapexpressionProject records into a new shape. Output keys are the keys of the expression.
--outjson | logfmt | csv | rawEncode each record. Default: matches input format.
--quietflagSuppress parse-error lines on stderr. Errors still set exit code on EOF.
--helpflagPrint usage and exit. Always exit 0.
